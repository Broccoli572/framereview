## Phase 1：联调与基础可用（P0）

### 1.1 部署验证
- [ ] 确认 Render 部署成功，前端页面可访问
- [ ] 确认 prisma db push 同步了新表（RefreshToken、PasswordReset）
- [ ] 确认 Express 正确 serve React SPA（SPA 路由正常工作）

### 1.2 核心流程联调
- [ ] 用户注册 → 登录 → JWT 获取

- [ ] 创建项目 → 创建文件夹 → 文件上传
- [ ] 素材列表 → 进入审片 → 添加批注
- [ ] 分享链接生成 → 访问分享页面

### 1.3 数据库 Seed
- [ ] 创建 `prisma/seed.js`：默认管理员账号 + 示例工作区/项目
- [ ] 首次部署自动执行 seed

### 1.4 错误处理统一
- [ ] 前端全局错误提示（Toast 通知）
- [ ] 401 自动跳转登录
- [ ] 网络错误友好提示
- [ ] API 错误响应格式统一验证

## Phase 2：媒体处理（P1）

### 2.1 Media Worker 服务
- [ ] 独立 Worker 进程（BullMQ 队列 + Redis）
- [ ] 视频转码（H.264 → 多码率 HLS）
- [ ] 缩略图提取（关键帧 × N 张）
- [ ] Sprite Strip 生成（时间轴预览条）
- [ ] 波形图提取（音频可视化）
- [ ] 视频元数据提取（时长/分辨率/帧率/编码）

### 2.2 上传体验
- [ ] 分段上传前端实现（大文件支持）
- [ ] 上传进度条 + 预估剩余时间
- [ ] 上传后自动触发 Media Worker 处理
- [ ] 处理状态实时展示（等待中/处理中/完成/失败）

### 2.3 对象存储集成
- [ ] S3/MinIO 接入（替代本地文件系统）
- [ ] 上传直传到 S3（减轻服务器负载）
